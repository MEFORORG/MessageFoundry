  SQL box:      SQL Server 2022 (RTM 16.0.1000.6), database `mfbench`, SQL login, RCSI = ON (verified;
                force-enabled at store open). Per handoff: i4i.2xlarge, store on local NVMe D:. It stayed

                was avoided.
  INTER-BOX RTT (disclosed per handoff §8): TCP-connect RTT engine->store (SQL box):1433, n=12:
                min 10.14 / median 11.38 / mean 12.68 / max 24.03 ms. This is HIGH for same-VPC EC2
                (usually sub-ms) and, at ~4-5 store round-trips per message, is a real co-bottleneck on
                the ABSOLUTE ceiling (~107 msg/s). It does NOT bias the A/B (both arms share the link, so
                the fusion LIFT is clean), but it is a plausible reason the lift is a modest ~10% rather
                than the projected large collapse — fusion cuts marshaling crossings, not network latency.
                A same-AZ / sub-ms rig might show a different (larger or smaller) fusion delta.
