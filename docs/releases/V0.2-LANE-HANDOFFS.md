# v0.2 Multisession — Lane Kickoff Prompts

Companion to [`MULTISESSION-PLAN-v0.2.md`](MULTISESSION-PLAN-v0.2.md) (the plan of record). These are
the **paste-ready first-prompt handoffs** for the one **coordinator** session and the **NOW** worker
lanes. Each begins with `ultracode` to enable UltraCode (it's session-only), and each tells the new
window to **spin up its own git worktree** before doing any work.

## How to use

Open a **new VS Code / Claude window per lane** and paste the matching prompt — the prompt instructs
the session to create its own worktree (`scripts\worktree\new.ps1 -Name <lane> [flags]`: a sibling
`..\MessageFoundry-<lane>` directory on branch `<lane>` off `origin/main`, with its own `.venv`) and
work only in it.

> If you'd rather have the window open **already inside** the worktree, run
> `scripts\worktree\spawn.ps1 -Name <lane> [flags]` first (it creates the worktree **and** opens a VS
> Code window in it), then paste the prompt there.

**Start order:** Lane X + Coordinator first, then A, then B / D / L in parallel. "Land first" for
Lane X is a *merge-order* constraint (its PR must merge before Lane B commits any `tests/conftest.py`
change), **not** a start-order one — spin up all the NOW windows together.

**Cross-cutting rules baked into every prompt:** each session works only in its own worktree (never
edits a sibling); branch → PR (never push to `main`); stage explicit paths (a hook blocks
`git add -A`); **only the Coordinator writes project memory** (workers read only); every PR runs the
gate (`ruff` + `ruff format --check`, `mypy` strict, `pytest` with `QT_QPA_PLATFORM=offscreen` for
console tests).

**Not started now:** Lane S (`store-eventlog`) and Lane Sec (`secrets-stepup`) are On-trigger/Then per
the plan. BACKLOG **#28 / #29** (load + throughput tests) are intentionally out of this plan — run them
ad hoc for a fresh baseline. **#24 DICOM** is deferred to v0.3 (so it stays on-trigger in Lane B).
