# `v0.1.0` — pre-tag checklist

> **Point-in-time checklist (2026-06-17).** What to run/verify **before** cutting the public `v0.1.0`
> tag. Tagging is **owner-driven and irreversible** (it triggers `release.yml` → first public PyPI
> publish of the final version). Nothing here cuts the tag; it's the sequence to follow when you decide to.
>
> **Mechanics (grounded):** the version is single-sourced as `__version__` in
> [`messagefoundry/__init__.py`](../../messagefoundry/__init__.py) (hatchling dynamic version);
> [`.github/workflows/release.yml`](../../.github/workflows/release.yml) fires on a `vX.Y.Z` tag push and
> runs build → wheel-smoke (**rejects a tag whose built version ≠ tag**) → CycloneDX SBOM → Sigstore
> keyless sign → SLSA build-provenance → GitHub release → **PyPI Trusted Publishing** (OIDC, no token).
