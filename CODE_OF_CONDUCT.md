# Code of Conduct

## Our standard

MessageFoundry adopts the **[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)**
as its code of conduct. We are committed to a welcoming, harassment-free experience for everyone who
takes part in the project, regardless of background or identity.

The full standard we operate under is the canonical Contributor Covenant v2.1, published at
<https://www.contributor-covenant.org/version/2/1/code_of_conduct/> (translations:
<https://www.contributor-covenant.org/translations/>). We adopt it **by reference** rather than
copying its text so we always track the canonical version.

## Scope

This Code applies in all project spaces — the repository and its issues, pull requests, discussions,
and review/commit comments — and whenever an individual is officially representing the project in a
public space.

## Reporting

If you experience or witness unacceptable behavior, report it **privately** to the project steward:

- a **[GitHub private security advisory](https://github.com/MEFORORG/MessageFoundry/security/advisories/new)**
  (routes privately to the maintainer), or
- the email listed on the maintainer's GitHub profile.

Do **not** open a public issue to report a Code of Conduct concern. All reports are reviewed promptly
and fairly, and the steward will respect the reporter's privacy and security.

## Enforcement

The project steward clarifies and enforces this Code and may take any action deemed appropriate — up
to a temporary or permanent ban from project spaces — following the Contributor Covenant's
**Enforcement Guidelines** (Correction → Warning → Temporary Ban → Permanent Ban). Until a second
maintainer joins (see [GOVERNANCE.md](GOVERNANCE.md)), the steward is the sole point of contact; this
single-point limitation is one we intend to resolve as the maintainer team grows.
